# terio

**Агрегатор интерфейсов. Все программы с CLI или API — из одной точки.**

terio — это терминал со встроенной AI-моделью. Вы говорите, что нужно сделать. terio строит цепочку команд, выполняет её и показывает результат. Удачная цепочка сохраняется как скрипт. В следующий раз тот же запрос выполняется без модели — мгновенно и дёшево.

## Для кого terio

Для тех, кто использует командную строку в терминале. Пользователи, привыкшие получать вывод после набора текста.

## Принцип

terio — агрегатор интерфейсов. Любая программа, с которой можно обменяться действием и результатом (через CLI, API, логи или иной канал), должна быть управляема из terio. Неважно, как предоставлен интерфейс. Важно, что есть принципиальная возможность передать программе описание действия и получить результат. terio обязан уметь интегрировать такую программу — то есть предоставить пользователю возможность управлять ею из единой точки.

Со временем terio научится работать со всеми программами, предоставляющими интерфейс. В MVP — через CLI.

## Как это работает

1. Вы вводите запрос: `terio ask "нарежь flac/cue на треки"`.
2. terio проверяет: **такой запрос уже был?** Если да — выполняет готовый скрипт. Модель не вызывается.
3. Если запрос новый — AI-модель (локальная или удалённая) строит цепочку команд.
4. terio показывает план, вы подтверждаете. Команды выполняются, результат рендерится.
5. Цепочка сохраняется как скрипт. Запрос помечается как «имеет дешёвое решение».
6. В следующий раз тот же запрос — скрипт, без модели.

**Scope расширяется лениво** — в процессе вашей обычной работы. terio не требует заранее подключать интеграции. Вы просто работаете, terio запоминает.

## Что terio снижает

1. **Стоимость внимания.** Все программы с CLI или API управляются из одной точки. Никаких переключений.
2. **Стоимость LLM.** Каждый повторяющийся запрос — это скрипт, а не токены.

## Логирование и метрики

terio логирует **всё, что печатает пользователь**. Это немного данных. terio считает **все доступные метрики** — счётчики выполнения, вызовы модели, попадания в кеш, длительность, ошибки. Это тоже немного.

Данные не агрегируются на лету — всё, что можно пересчитать из лога, пересчитывается по запросу. Главное — ничего не терять из поступающей статистики.
