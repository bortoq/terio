# terio

**Панель управления, которая учится на ваших действиях.**

terio — это терминал со встроенной AI-моделью. Вы говорите, что нужно сделать. terio строит цепочку команд, выполняет её и показывает результат. Удачная цепочка сохраняется как скрипт. В следующий раз тот же запрос выполняется без модели — мгновенно и дёшево.

## Как это работает

1. Вы вводите запрос: `terio ask "нарежь flac/cue на треки и переименуй как в прошлый раз"`.
2. terio проверяет: **такой запрос уже был?** Если да — выполняет готовый скрипт. Модель не вызывается.
3. Если запрос новый — AI-модель (локальная или удалённая) строит цепочку shell-команд.
4. terio показывает план, вы подтверждаете. Команды выполняются, результат рендерится.
5. Цепочка сохраняется как скрипт. Запрос помечается как «имеет дешёвое решение».
6. В следующий раз тот же запрос — скрипт, без модели.

**Scope расширяется лениво** — в процессе вашей обычной работы. terio не требует заранее подключать интеграции. Вы просто работаете, terio запоминает.

## Какие ценности снижает terio

1. **Стоимость внимания.** Все программы с CLI или API управляются из одной точки. Никаких переключений между терминалом, браузером, плеером и CI-дашбордом.
2. **Стоимость LLM.** Каждый повторяющийся запрос — это скрипт, а не токены. Модель вызывается только когда задача встречается впервые.

## Для кого terio

Для всех, кто работает в командной строке: разработчики, DevOps, медиа-пользователи, админы. terio встаёт на место эмулятора терминала и добавляет AI-помощника, который учится на ваших повторяющихся действиях.
